Using console.log() for Debugging

This page demonstrates various ways to use the console for debugging JavaScript. Open your browser's console (F12 or Ctrl+Shift+J) to see the output.

1. Basic console.log()

The simplest use of console.log() to output values:

console.log("Hello, World!");
console.log(42);
console.log(true);

2. Logging Variables and Objects

Debugging variables and inspecting objects:

let person = {
    name: "John Doe",
    age: 30,
    job: "Developer"
};

console.log("Person object:", person);
console.log("Name:", person.name);
console.log("Age:", person.age);

3. Debugging Function Execution

Using console.log() to track function execution and parameter values:

function calculateArea(width, height) {
    console.log("Function calculateArea called");
    console.log("Parameters:", { width, height });
    
    const area = width * height;
    console.log("Calculated area:", area);
    
    return area;
}

const area = calculateArea(5, 10);

4. Advanced Console Methods

Beyond console.log() - Other useful methods:

console.info("This is information");
console.warn("This is a warning");
console.error("This is an error");
console.table([
    { name: "John", age: 30 },
    { name: "Jane", age: 25 }
]);

5. Debugging with Timers

Measuring execution time:

console.time("Loop time");
for (let i = 0; i < 1000000; i++) {
    // Some operation
}
console.timeEnd("Loop time");

6. Debugging a Bug

Using console.log() to find and fix a bug:

function calculateSum(numbers) {
    console.log("Input array:", numbers);
    
    let sum = 0;
    for (let i = 0; i < numbers.length; i++) {
        console.log(`Adding numbers[${i}]: ${numbers[i]}`);
        sum += numbers[i];
        console.log(`Current sum: ${sum}`);
    }
    
    return sum;
}

Tips for Effective Console Debugging:

  • Use descriptive labels in your console.log() messages
  • Use console.table() for arrays and objects when appropriate
  • Use console.time() and console.timeEnd() to measure performance
  • Use console.warn() and console.error() for different severity levels
  • Remember to remove or disable console.log() statements in production code
